What Phlux is
  • Software-first inference of key operating variables when direct measurement is costly or impractical.
  • Reusable approach across assets when system physics are similar.
  • Built by engineers for teams that need traceability, robustness, and real-world performance.
Typical outputs
  • Flow rates (oil / gas / water or single-phase)
  • Equipment condition indicators (e.g., degradation proxies)
  • Operational state estimates and alerts

What we do

Phlux focuses on delivering actionable estimates from limited signals, using models that are designed to be validated and deployed.

Infer what you can’t measure

Estimate key variables using available telemetry, boundary conditions, and system knowledge — without adding hardware at every location.

Blend physics with learning

Use physics-based simulation to structure the problem and generate training data, then apply ML for fast, robust inference in operation.

Deploy like a product

Package models with preprocessing, schemas, and versioning so they can run reliably in real workflows (edge or cloud).

How it works

A simple, repeatable lifecycle: generate / validate → train → deploy → monitor.

Step 1
Define the scope

Signals available, outputs required, and operating envelope.

Step 2
Build + validate

Use physics-based modeling and field data alignment where available.

Step 3
Train for inference

Train ML models on synthetic + historical data with realistic noise and constraints.

Step 4
Deploy + monitor

Package the runtime with versioning, tests, and monitoring for drift and reliability.

Use cases

Example applications — Phlux is designed to extend measurement and improve operational awareness.

Virtual Flow Metering

Estimate multiphase or single-phase flow rates using pressure/temperature and other available signals, reducing dependency on inline meters.

Equipment condition inference

Track condition indicators (e.g., degradation proxies) from system response, enabling earlier detection of performance loss.

Operational state + alerts

Detect off-normal behavior and provide state estimates that support troubleshooting and decision-making.
